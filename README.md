# Лабораторная работа 3: Многопоточная система управления лифтами

Часть 1. Реализовано:
- Класс лифта с основным циклом движения
- Обработка внутренних целей
- Диспетчер с выбором подходящего лифта
- Потокобезопасная очередь запросов
- Консольный интерфейс для отправки запросов

Часть 1 является минимально работоспособной симуляцией без продвинутой оптимизации и без ограничения пассажиров.

Часть 2. Реализовано:
- Поток лифта работает постоянно, обрабатывая назначения и открывая/закрывая двери
- Класс Request с уникальными ID для каждого запроса
- Сохранение и загрузка состояния системы в файл и из файла
- Автоматический диспетчер распределяет запросы между лифтами по ближайшему этажу
- Поддержка нескольких лифтов (настраивается в Main)
- Автоматический и ручной тестирование работы системы, включая проверку конкуренции и устойчивости к нагрузке
- Команды CLI: <этаж> <up|down>, status, save <файл>, load <файл>, exit

Часть 2 расширяет функциональность симуляции, делает её устойчивой к множественным запросам и позволяет сохранять текущее состояние для продолжения работы.
